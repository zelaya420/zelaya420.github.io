## Bandit – Nivel 4 → Nivel 5
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra almacenada en el único archivo **legible por humanos** dentro del directorio `inhere` del usuario **bandit4**.

La dificultad principal consiste en distinguir, entre varios archivos con nombres similares, cuál contiene texto legible y cuál contiene datos binarios o contenido no interpretable directamente.  
Por ello, el objetivo es enumerar los archivos disponibles, identificar su tipo y leer únicamente el archivo correcto.

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:


- `ls` - `ls` – Mostrar el contenido de un directorio  
- `cd` – Desplazarse entre directorios  
- `cat` – Visualizar el contenido de archivos  
- `file` – Obtener información sobre el tipo de archivo  
- `du` – Ver el tamaño de archivos o directorios  
- `find` – Localizar archivos dentro del sistema  

Aunque el nivel menciona varios comandos de apoyo, la resolución puede hacerse de forma limpia usando principalmente `ls`, `cd`, `file` y `cat`.

## Conexión al servidor (Nivel 4)

Para acceder al Nivel 4 de Bandit, nos autenticamos por SSH con el usuario correspondiente a este nivel:

```javascript
sshpass -p '2WmrDFRmJIq3IPxneAaMGhap0pFhF3NJ' ssh bandit4@bandit.labs.overthewire.org -p 2220
```
El uso de sshpass permite pasar la contraseña directamente en el comando, evitando la entrada interactiva y facilitando conexiones automatizadas en un entorno de práctica.

Una vez establecida la conexión, el sistema mostrará el banner de bienvenida del wargame, confirmando que nos encontramos en el entorno correspondiente al Nivel 4.

Ajuste del entorno de terminal

Tras iniciar sesión, se puede definir la variable de entorno TERM para asegurar una mejor compatibilidad con la terminal:

```javascript
export TERM=xterm
```

Esto no es estrictamente obligatorio para resolver el nivel, pero puede ayudar a evitar comportamientos extraños en algunos entornos de terminal o máquinas virtuales.

Enumeración del directorio

Al iniciar sesión, nos encontramos en el directorio personal del usuario bandit4.
Comenzamos enumerando su contenido:

```javascript
ls
```

La salida muestra un único directorio relevante:

```javascript
inhere
```

Entramos en él para inspeccionar los archivos contenidos:

```javascript
cd inhere/
```

Después listamos su contenido:

```javascript
ls
```
La salida muestra varios archivos con nombres similares:

```javascript
-file00  -file01  -file02  -file03  -file04  -file05  -file06  -file07  -file08  -file09
```

¿Cuál es el problema en este nivel?

A simple vista, todos los archivos tienen nombres parecidos y ninguno revela directamente cuál contiene la contraseña.
Si intentáramos abrirlos uno por uno con cat, estaríamos trabajando por prueba y error, lo cual no es una buena práctica.

La idea correcta es caracterizar primero los archivos antes de leerlos.

Aquí entra en juego el comando file, que permite identificar el tipo de contenido de cada archivo.

Identificación del archivo correcto

Para analizar todos los archivos del directorio actual, utilizamos:

```javascript
file ./*
```
La salida obtenida es:

```javascript
./-file00: data
./-file01: data
./-file02: data
./-file03: data
./-file04: data
./-file05: data
./-file06: data
./-file07: ASCII text
./-file08: data
./-file09: data
```
¿Por qué usar file ./*?

Este comando es clave por varias razones:

file inspecciona el contenido del archivo, no solo su nombre
./* expande todos los archivos del directorio actual
permite identificar rápidamente cuál tiene texto legible por humanos
evita abrir archivos binarios innecesariamente

En este caso, la mayoría de los archivos aparecen como data, es decir, datos sin una estructura textual útil a simple vista.
Sin embargo, uno de ellos destaca claramente:

```javascript
./-file07: ASCII text
```

Esto indica que -file07 contiene texto plano legible, por lo que es el candidato correcto para almacenar la contraseña.

Obtención de la contraseña del Nivel 5

Una vez identificado el archivo correcto, procedemos a leer su contenido:

```javascript
cat ./-file07
```

La salida revela la contraseña del usuario bandit5:

```javascript
4oQYVPkxZOOEOO5pTW81FB8j8lxXGUQw
```
Conclusión del nivel

Este nivel introduce un concepto muy importante en Linux y en análisis técnico en general:
antes de interactuar con múltiples archivos, conviene identificarlos y clasificarlos.

La solución refuerza varios puntos fundamentales:

la importancia de enumerar antes de actuar
el uso de file para reconocer tipos de archivo
la diferencia entre datos arbitrarios y texto legible
el manejo correcto de archivos cuyos nombres comienzan con -

Más allá de obtener la contraseña, este nivel enseña una metodología más sólida:
enumerar, identificar, filtrar y luego leer.

Con la contraseña obtenida, ya es posible autenticarse como bandit5 y continuar con el Nivel 5 → Nivel 6.
