## Bandit – Nivel 10 → Nivel 11
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra almacenada en el archivo `data.txt`, codificada en **Base64**.

La dificultad principal consiste en reconocer que el contenido del archivo no es una contraseña en texto plano, sino una representación codificada que debe decodificarse para obtener el valor real.

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `cat` – Visualizar el contenido de archivos  
- `base64` – Codificar y decodificar datos en Base64  
- `tr` – Reemplazar o transformar caracteres  
- `grep` – Buscar patrones de texto  
- `xxd` – Convertir entre hexadecimal y binario  

En este nivel, el comando clave es `base64`.

## Conexión al servidor (Nivel 10)

Para acceder al Nivel 10 de Bandit, nos autenticamos por SSH con el usuario correspondiente a este nivel:

```javascript
sshpass -p 'FGUW5ilLVJrxX9kMYMmlN4MgbpfMiqey' ssh bandit10@bandit.labs.overthewire.org -p 2220
```

El uso de sshpass permite pasar la contraseña directamente en el comando, evitando la entrada interactiva y facilitando conexiones automatizadas en un entorno de práctica.

Una vez establecida la conexión, el sistema mostrará el banner de bienvenida del wargame, confirmando que nos encontramos en el entorno correspondiente al Nivel 10.

Enumeración inicial

Al iniciar sesión, verificamos el contenido del directorio personal:
```javascript
ls
```

La salida muestra:

```javascript
data.txt
```


Esto indica que el archivo relevante para el nivel ya está disponible directamente en el directorio actual.

Inspección del archivo

Visualizamos el contenido de data.txt con:

```javascript
cat data.txt
```

La salida fue:

```javascript
VGhlIHBhc3N3b3JkIGlzIGR0UjE3M2ZaS2IwUlJzREZTR3NnMlJXbnBOVmozcVJyCg==
```

A primera vista, esto no parece una contraseña directa.
El contenido tiene el patrón típico de una cadena en Base64:

usa caracteres alfanuméricos
incluye +, / o = como posibles caracteres de padding
suele terminar en = o ==

En este caso, la presencia de == al final es una señal fuerte de que se trata de texto codificado en Base64.

Decodificación del contenido

Para obtener el valor original, ejecutamos:

```javascript
base64 --decode data.txt
```

La salida obtenida fue:

```javascript
The password is dtR173fZKb0RRsDFSGsg2RWnpNVj3qRr
```

¿Por qué funciona base64 --decode?

Base64 es un esquema de codificación que transforma datos binarios o texto en caracteres ASCII imprimibles.

No es cifrado ni hash.
Eso significa que no hay que “romper” nada: simplemente hay que decodificar el contenido para recuperar el texto original.

El comando:

```javascript
base64 --decode data.txt
```

lee el contenido codificado del archivo y lo transforma de vuelta a su representación original.

Identificación de la contraseña

La salida decodificada contiene una frase completa:

```javascript
The password is dtR173fZKb0RRsDFSGsg2RWnpNVj3qRr
```

Aquí es importante no cometer un error de interpretación:
la contraseña no es toda la frase, sino únicamente el valor final.

La contraseña del usuario bandit11 es:

```javascript
dtR173fZKb0RRsDFSGsg2RWnpNVj3qRr
```

Error común en este nivel

Un error frecuente consiste en asumir que la cadena vista con cat ya es la contraseña:

```javascript
VGhlIHBhc3N3b3JkIGlzIGR0UjE3M2ZaS2IwUlJzREZTR3NnMlJXbnBOVmozcVJyCg==
```

Eso es incorrecto.
Ese valor es solo la representación codificada.

Otro error común es pensar que Base64 “protege” la contraseña como si fuera cifrado.
No es así: Base64 solo cambia la forma del texto para que sea transportable o legible en ASCII.

La lección aquí es clara: codificación no es cifrado.

Conclusión del nivel

Este nivel introduce una habilidad básica pero muy importante en Linux y en seguridad informática:
reconocer y decodificar contenido representado en Base64.

La solución refuerza varios puntos clave:

identificar patrones típicos de Base64
distinguir codificación de cifrado
usar base64 --decode para recuperar el contenido original
interpretar correctamente la salida para extraer solo la contraseña

Más allá de obtener la contraseña, este nivel enseña una idea fundamental que aparece constantemente en logs, tokens, payloads y datos transportados entre sistemas:
muchos datos no están cifrados, solo codificados.
