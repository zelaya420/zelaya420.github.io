## Bandit – Nivel 1 → Nivel 2
Objetivo del nivel

En este desafío, la contraseña necesaria para avanzar al siguiente nivel se encuentra dentro de un archivo ubicado en el directorio home del usuario. La particularidad de este archivo es que su nombre utiliza un carácter especial, lo que puede dificultar su acceso si no se emplean los comandos de forma correcta.

El objetivo principal es identificar y leer un archivo cuyo nombre comienza con un guion (-), entendiendo cómo la terminal interpreta este tipo de nombres y cómo evitar errores comunes al interactuar con ellos.

Este nivel refuerza el manejo de archivos con nombres poco convencionales y el uso adecuado de la línea de comandos en Linux.


## Comandos útiles
Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `ls` – Mostrar el contenido de un directorio  
- `cd` – Desplazarse entre directorios  
- `cat` – Visualizar el contenido de archivos  
- `file` – Obtener información sobre el tipo de archivo  
- `du` – Ver el tamaño de archivos o directorios  
- `find` – Localizar archivos dentro del sistema  

find – Localizar archivos dentro del sistema

Estos comandos son parte fundamental del entorno Linux y se utilizarán de forma recurrente a lo largo de los siguientes niveles del wargame.

## Conexión al servidor (Nivel 1)
Para acceder al Nivel 1 de Bandit, utilizamos nuevamente una conexión por SSH, esta vez autenticándonos con el usuario correspondiente al nivel actua

```javascript
❯ ssh bandit1@bandit.labs.overthewire.org -p 2220
```
![Bandit Nivel 1 – 2s](/assets/img/Bandit/Bandit1.jpeg)

Cuando el sistema solicite la contraseña, se debe ingresar la credencial obtenida en el nivel anterior.

Una vez establecida la conexión, estaremos ubicados en el directorio personal del usuario bandit1, desde donde podremos comenzar la enumeración necesaria para localizar la contraseña del Nivel 2.

## Enumeración del directorio

Una vez conectados como el usuario bandit1, comenzamos la enumeración del directorio personal utilizando el siguiente comando:


```javascript
ls -la
```
![Bandit Nivel 1 – 2s](/assets/img/Bandit/Bandit01.png)

Este comando permite listar todos los archivos del directorio, incluyendo los archivos ocultos, y muestra información detallada como permisos, propietario, grupo y tamaño.

Al ejecutar el comando, observamos la presencia de un archivo cuyo nombre es únicamente un guion (-), el cual contiene la contraseña necesaria para avanzar al siguiente nivel.
### ¿Por qué usar ls -la?

El uso de ls -la es clave en este nivel porque:

-l muestra información detallada de cada archivo

-a incluye archivos ocultos y archivos con nombres especiales

Permite identificar propietarios y permisos, lo que confirma si el archivo es accesible

En este caso, podemos ver que el archivo - pertenece al usuario correspondiente y cuenta con permisos de lectura, por lo que es posible visualizar su contenido.

## Obtención de la contraseña del Nivel 2

Después de identificar el archivo con nombre especial (-) y comprender por qué no puede leerse directamente con cat "-", procedemos a acceder a él especificando su ruta relativa.

```javascript
cat ./-
```

Al ejecutar este comando, el sistema muestra correctamente el contenido del archivo, revelando la contraseña necesaria para acceder al siguiente nivel.
```javascript
263JGJPfgU6LtDEvgfWU1XP5yac29mFx
```

![Bandit Nivel 1 – 2s](/assets/img/Bandit/Bandit001.png)

## Conclusión del nivel

Este nivel introduce un concepto clave en el uso de la línea de comandos: el manejo de archivos cuyos nombres comienzan con caracteres especiales. Comprender cómo los comandos interpretan sus argumentos es fundamental para evitar comportamientos inesperados y para interactuar correctamente con el sistema.

La solución refuerza la importancia de la enumeración detallada y del uso consciente de rutas relativas al trabajar en entornos Linux.

Con la contraseña obtenida, ya es posible autenticarse como bandit2 y continuar con el Nivel 2 → Nivel 3.
