## Bandit – Nivel 7 → Nivel 8
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra almacenada dentro del archivo `data.txt`, junto a la palabra **millionth**.

La dificultad principal consiste en localizar una línea específica dentro de un archivo de texto y extraer de ella el valor relevante.  
Por ello, el objetivo es identificar rápidamente la línea que contiene la palabra clave y obtener la contraseña asociada.

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `grep` – Buscar texto dentro de archivos  
- `cat` – Visualizar el contenido de archivos  
- `head` – Mostrar las primeras líneas de un archivo  
- `tail` – Mostrar las últimas líneas de un archivo  
- `sort` – Ordenar líneas de texto  
- `uniq` – Filtrar líneas repetidas  
- `strings` – Extraer texto legible desde datos binarios  

En este nivel, el comando más importante es `grep`, ya que permite buscar directamente la palabra clave dentro del archivo.

Búsqueda de la línea correcta

Sabemos por el enunciado que la contraseña está almacenada en el archivo data.txt, junto a la palabra millionth.
Por lo tanto, la estrategia correcta es buscar esa palabra dentro del archivo.

Un intento inicial puede ser:

```javascript
grep "millionth" data.txt | cut -d ' ' -f2
```

La salida obtenida fue:

```javascript
millionth	dfwvzFQi4mU0wfNbFOe9RoWskMLg7eEc
```

¿Por qué grep funciona?

El comando:

```javascript
grep "millionth" data.txt
```

busca dentro de data.txt cualquier línea que contenga la cadena millionth.

En este nivel, esa palabra aparece junto a la contraseña, por lo que encontrar la línea correcta es suficiente para resolver el desafío.

¿Qué problema hubo con cut?

Aunque el pipeline anterior devolvió la línea correcta, cut no separó la salida como esperabas.

```javascript
cut -d ' ' -f2
```

asume que el delimitador entre columnas es un espacio normal.
Sin embargo, en este archivo la separación parece estar hecha con una tabulación, no con un espacio simple.

Por eso, en lugar de extraer únicamente la contraseña, la salida conservó toda la línea.

En otras palabras:

grep sí encontró la línea correcta
cut no interpretó bien el separador
Forma correcta de obtener la contraseña

La opción más simple es mostrar directamente la línea:

```javascript
grep "millionth" data.txt
```

Eso devolverá algo como:

```javascript
millionth	dfwvzFQi4mU0wfNbFOe9RoWskMLg7eEc
```

Y de ahí se identifica la contraseña como el segundo campo.

Si se quiere extraer únicamente la contraseña de forma robusta, una mejor opción es:

```javascript
grep "millionth" data.txt | awk '{print $2}'
```

Esto funciona porque awk separa campos por espacios en blanco de manera más flexible, incluyendo espacios y tabulaciones.

Obtención de la contraseña del Nivel 8

La contraseña del usuario bandit8 es:

```javascript
dfwvzFQi4mU0wfNbFOe9RoWskMLg7eEc
```

Error común en este nivel

Un error frecuente consiste en usar herramientas de corte sin comprobar primero cómo está delimitado el texto.

Por ejemplo:

```javascript
grep "millionth" data.txt | cut -d ' ' -f2
```

puede fallar si el archivo usa tabs en vez de espacios.

La lección aquí es clara: antes de asumir cómo está estructurado un archivo, hay que considerar que los separadores pueden no ser visibles a simple vista.

Conclusión del nivel

Este nivel introduce una habilidad fundamental en Linux:
la búsqueda eficiente de texto dentro de archivos usando grep.

La solución refuerza varios puntos importantes:

localizar texto clave dentro de archivos grandes
usar grep como herramienta de filtrado rápido
no asumir que los delimitadores son siempre espacios
elegir herramientas adecuadas para extraer columnas, como awk

Más allá de obtener la contraseña, este nivel enseña un patrón muy común en terminal y análisis de datos:
buscar primero la línea correcta y luego extraer el campo correcto.

Con la contraseña obtenida, ya es posible autenticarse como bandit8 y continuar con el Nivel 8 → Nivel 9.
