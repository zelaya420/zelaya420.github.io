## Bandit – Nivel 9 → Nivel 10
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra en el archivo `data.txt`, dentro de una de las pocas cadenas legibles por humanos, precedida por varios caracteres `=`.

La dificultad principal consiste en que `data.txt` no es un archivo de texto plano convencional, por lo que no conviene intentar leerlo directamente con `cat`.  
El objetivo es extraer las cadenas legibles del archivo y localizar aquella que esté precedida por varios signos `=`.

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `strings` – Extraer cadenas legibles desde archivos binarios o con datos no imprimibles  
- `grep` – Buscar patrones dentro de texto  
- `sort` – Ordenar líneas de texto  
- `uniq` – Filtrar líneas repetidas  
- `base64` – Codificar o decodificar Base64  
- `tr` – Reemplazar o eliminar caracteres  
- `tar`, `gzip`, `bzip2` – Manipular archivos comprimidos  
- `xxd` – Convertir entre hexadecimal y binario  

En este nivel, los comandos clave son `strings` y `grep`.

Extracción de cadenas legibles

Sabemos por el enunciado que la contraseña está en data.txt, dentro de una de las pocas cadenas legibles por humanos y precedida por varios signos =.

Para resolverlo, ejecutamos:

```javascript
strings data.txt | grep "^==="
```
La salida obtenida es:
```javascript
========== password
========== is
========== FGUW5ilLVJrxX9kMYMmlN4MgbpfMiqey
```

¿Por qué funciona strings?

El comando strings extrae secuencias de caracteres imprimibles desde archivos que contienen datos binarios o no legibles directamente.

Esto es útil porque data.txt no está pensado para ser interpretado como texto plano completo.
Si se usara cat, la salida probablemente sería confusa o incluiría caracteres no útiles.

Con strings, reducimos el archivo a las partes que sí contienen texto legible por humanos.

¿Por qué funciona grep "^==="?

El comando:
```javascript
grep "^==="
```

filtra únicamente las líneas que comienzan con varios signos =.

Desglose del patrón
^ indica el inicio de la línea
=== exige que la línea comience con al menos esos tres caracteres =

Esto es coherente con el enunciado, que dice que la contraseña está precedida por varios caracteres =.

Al combinar ambos comandos:

```javascript
strings data.txt | grep "^==="
```

primero se extraen cadenas legibles y después se filtran solo aquellas que cumplen el patrón esperado.

Identificación de la contraseña

La salida contiene tres líneas:

```javascript
========== password
========== is
========== FGUW5ilLVJrxX9kMYMmlN4MgbpfMiqey
```

Aquí hay que evitar un error de cierre prematuro: no toda línea con = es la contraseña.

Las dos primeras son solo texto contextual:
```javascript
========== password
========== is
```

La tercera línea contiene el valor real que buscamos:
```javascript
========== FGUW5ilLVJrxX9kMYMmlN4MgbpfMiqey
```

Por lo tanto, la contraseña del usuario bandit10 es:
```javascript
FGUW5ilLVJrxX9kMYMmlN4MgbpfMiqey
```

Error común en este nivel


Un error frecuente sería intentar leer el archivo directamente con:

```javascript
cat data.txt
```

Eso no aprovecha la pista del nivel y puede producir una salida poco útil si el archivo contiene muchos datos no imprimibles.

Otro error sería usar un grep demasiado general, por ejemplo:

```javascript
strings data.txt | grep "="
```

Eso puede devolver más líneas de las necesarias y aumentar el ruido.

La lección aquí es clara: cuando el enunciado ya describe un patrón concreto, conviene filtrar con precisión.

Conclusión del nivel

Este nivel introduce una habilidad importante en Linux:
extraer información relevante desde archivos que no son texto plano convencional.

La solución refuerza varios puntos clave:

usar strings para recuperar texto legible desde datos binarios
aplicar grep con patrones precisos
aprovechar las pistas del enunciado para reducir ruido
distinguir entre texto contextual y el valor realmente útil

Más allá de obtener la contraseña, este nivel enseña un patrón muy común en análisis técnico:
primero extraer contenido interpretable y después filtrar por el patrón correcto.
