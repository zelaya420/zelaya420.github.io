## Bandit – Nivel 2 → Nivel 3
Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra en un archivo ubicado en el directorio home del usuario. La particularidad de este archivo es que su nombre contiene espacios, lo que requiere un manejo correcto desde la línea de comandos para poder acceder a él.

El objetivo es identificar y leer correctamente un archivo con espacios en su nombre, comprendiendo cómo Bash interpreta este tipo de rutas.

## Comandos útiles
Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `ls` – Mostrar el contenido de un directorio  
- `cd` – Desplazarse entre directorios  
- `cat` – Visualizar el contenido de archivos  
- `file` – Obtener información sobre el tipo de archivo  
- `du` – Ver el tamaño de archivos o directorios  
- `find` – Localizar archivos dentro del sistema  

find – Localizar archivos dentro del sistema

Estos comandos son parte fundamental del entorno Linux y se utilizarán de forma recurrente a lo largo de los siguientes niveles del wargame.

## Conexión al servidor (Nivel 2)
Para acceder al Nivel 2 de Bandit, utilizamos nuevamente una conexión por SSH, esta vez autenticándonos con el usuario correspondiente al nivel actua

```javascript
sshpass -p '263JGJPfgU6LtDEvgfWU1XP5yac29mFx' ssh bandit2@bandit.labs.overthewire.org -p 2220
```
![Bandit Nivel 1 – 2s](/assets/img/Bandit/Bandit2.png)

Para evitar introducir la contraseña de forma interactiva, podemos utilizar la herramienta sshpass, la cual permite pasar la credencial directamente en el comando y establecer la conexión de manera automática.

Al ejecutar este comando, se establece correctamente la conexión con el servidor de OverTheWire Bandit, autenticándonos como el usuario bandit2 sin necesidad de ingresar la contraseña manualmente.

Una vez completado el acceso, el sistema muestra el banner de bienvenida del wargame, confirmando que nos encontramos dentro del entorno correspondiente al Nivel 2, listos para continuar con la enumeración y resolver el siguiente desafío.

Una vez establecida la conexión, estaremos ubicados en el directorio personal del usuario bandit2, desde donde podremos comenzar la enumeración necesaria para localizar la contraseña del Nivel 3.

## Enumeración del directorio

Una vez conectados como el usuario bandit1, comenzamos la enumeración del directorio personal utilizando el siguiente comando:


```javascript
ls -la
```
![Bandit Nivel 1 – 2s](/assets/img/Bandit/Bandit02.png)

## ¿Por qué repetir el mismo comando?

Este comando permite listar todos los archivos del directorio, incluyendo los archivos ocultos, y muestra información detallada como permisos, propietario, grupo y tamaño.

Al ejecutar el comando, observamos la presencia de un archivo cuyo nombre es únicamente un guion (-), el cual contiene la contraseña necesaria para avanzar al siguiente nivel.

-l muestra información detallada de cada archivo

-a incluye archivos ocultos y archivos con nombres especiales

Permite identificar propietarios y permisos, lo que confirma si el archivo es accesible

En este caso, podemos ver que el archivo - pertenece al usuario correspondiente y cuenta con permisos de lectura, por lo que es posible visualizar su contenido.

## Obtención de la contraseña del Nivel 

```javascript
cat ./--spaces\ in\ this\ filename--
```

Este comando muestra el contenido del archivo, revelando la contraseña necesaria para avanzar al siguiente nivel.

```javascript
cat ./--spaces\ in\ this\ filename--
```
En Linux, cualquier argumento que comienza con `-` es interpretado como una opción del comando.  
Al anteponer `./`, indicamos explícitamente que se trata de un archivo ubicado en el directorio actual, evitando que `cat` lo interprete como una opción.

```javascript
MNk8KNH3Usiio41PRUEoDFPqfxLPlSmx
```

## Conclusión del nivel

Este nivel introduce un concepto clave en el uso de la línea de comandos: el manejo de archivos cuyos nombres comienzan con caracteres especiales y espacios en el nombre.Comprender cómo los comandos interpretan sus argumentos es fundamental para evitar comportamientos inesperados y para interactuar correctamente con el sistema.

La solución refuerza la importancia de la enumeración detallada y del uso consciente de rutas relativas al trabajar en entornos Linux.

Con la contraseña obtenida, ya es posible autenticarse como bandit3 y continuar con el Nivel 3 → Nivel 4.
