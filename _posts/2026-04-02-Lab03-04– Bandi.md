## Bandit – Nivel 3 → Nivel 4
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra almacenada en un **archivo oculto** dentro del directorio `inhere` del usuario **bandit3**.

La dificultad principal consiste en comprender cómo Linux maneja los archivos ocultos: cualquier archivo cuyo nombre comienza con un punto (`.`) no se muestra con un `ls` normal.  
Por ello, el objetivo es identificar correctamente el directorio indicado, listar su contenido incluyendo archivos ocultos y leer el archivo que contiene la contraseña.

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `ls` – Mostrar el contenido de un directorio  
- `cd` – Desplazarse entre directorios  
- `cat` – Visualizar el contenido de archivos  
- `file` – Obtener información sobre el tipo de archivo  
- `du` – Ver el tamaño de archivos o directorios  
- `find` – Localizar archivos dentro del sistema  

Aunque varios comandos aparecen como apoyo, para este nivel basta con una enumeración básica del directorio y una lectura correcta del archivo oculto.

## Conexión al servidor (Nivel 3)

Para acceder al Nivel 3 de Bandit, nos autenticamos por SSH con el usuario correspondiente a este nivel:

```javascript
sshpass -p 'MNk8KNH3Usiio41PRUEoDFPqfxLPlSmx' ssh bandit3@bandit.labs.overthewire.org -p 2220
```

El uso de sshpass permite pasar la contraseña directamente en el comando, evitando la entrada interactiva y facilitando conexiones automatizadas en un entorno de práctica.

Una vez establecida la conexión, el sistema mostrará el banner de bienvenida del wargame, confirmando que nos encontramos en el entorno correspondiente al Nivel 3.

Enumeración del directorio

Al iniciar sesión, nos encontramos en el directorio personal del usuario bandit3.
Comenzamos enumerando su contenido:

```javascript
ls
```
La salida muestra un único directorio relevante:
```javascript
inhere
```
Entramos en él para inspeccionar su contenido:

```javascript
cd inhere
```
A continuación, listamos todos los archivos, incluyendo los ocultos:
```javascript
ls -la
```

¿Por qué usar ls -la?

Este comando permite listar todos los archivos del directorio, incluidos los ocultos, y además muestra información detallada como permisos, propietario, grupo y tamaño.

- `-l` muestra información detallada de cada archivo
- `-a` incluye archivos ocultos (los que comienzan con .)

Al ejecutar el comando, observamos el archivo:
```javascript
...Hiding-From-You
```
Este archivo comienza con puntos, por lo que no sería visible con un ls normal.
Aquí es importante no asumir que el nombre será .hidden o algo parecido: el nombre exacto debe copiarse tal como aparece en la salida.

Obtención de la contraseña del Nivel 4

Una vez identificado el archivo, podemos leer su contenido con:

```javascript
cat ...Hiding-From-You
```
También es válido usar una ruta explícita:

```javascript
cat "./...Hiding-From-You"
```

Este comando muestra el contenido del archivo, revelando la contraseña necesaria para avanzar al siguiente nivel.

Obtención de la contraseña

Al ejecutar el comando, se muestra la contraseña del usuario bandit4:

```javascript
2WmrDFRmJIq3IPxneAaMGhap0pFhF3NJ
```
Error común en este nivel

Un error frecuente consiste en escribir mal el nombre del archivo, por ejemplo:

```javascript
cat .Hiding-From-You
```

Esto falla porque el archivo no comienza con un solo punto, sino con tres puntos:

```javascript
...Hiding-From-You
```

La lección aquí es clara: en Linux, cualquier archivo que comience con . es oculto, pero puede tener uno o varios puntos al inicio.
No conviene asumir el nombre; hay que leerlo exactamente como aparece en la enumeración.

Conclusión del nivel

Este nivel introduce un concepto esencial del uso de Linux en terminal:
la diferencia entre archivos visibles y archivos ocultos.

La solución refuerza varios puntos importantes:

la necesidad de enumerar correctamente antes de actuar
el uso de ls -la para descubrir archivos ocultos
la importancia de copiar con precisión los nombres de archivo
evitar suposiciones prematuras sobre nombres o rutas

Con la contraseña obtenida, ya es posible autenticarse como bandit4 y continuar con el Nivel 4 → Nivel 5.
