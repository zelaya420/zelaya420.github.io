## Bandit – Nivel 11 → Nivel 12
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra almacenada en el archivo `data.txt`, pero ha sido codificada usando **ROT13**.

La dificultad principal consiste en reconocer que el contenido no está cifrado con un algoritmo complejo, sino transformado mediante una rotación alfabética simple.  
El objetivo es revertir esa transformación para recuperar la contraseña real.

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `ls` – Mostrar el contenido de un directorio  
- `cat` – Visualizar el contenido de archivos  
- `tr` – Reemplazar o transformar caracteres  
- `grep` – Buscar patrones de texto  
- `xxd` – Convertir entre hexadecimal y binario  
- `base64` – Codificar o decodificar Base64  

En este nivel, el comando clave es `tr`.

## Conexión al servidor (Nivel 11)

```javascript
sshpass -p 'dtR173fZKb0RRsDFSGsg2RWnpNVj3qRr' ssh bandit11@bandit.labs.overthewire.org -p 2220
```

El uso de sshpass permite pasar la contraseña directamente en el comando, evitando la entrada interactiva y facilitando conexiones automatizadas en un entorno de práctica.

Una vez establecida la conexión, el sistema mostrará el banner de bienvenida del wargame, confirmando que nos encontramos en el entorno correspondiente al Nivel 11.

Enumeración inicial

Al iniciar sesión, verificamos el contenido del directorio personal:

```javascript
ls
```

La salida muestra:

```javascript
data.txt
```

Esto confirma que el archivo relevante para el nivel ya está disponible directamente en el directorio actual.

Inspección del archivo

Antes de aplicar cualquier transformación, conviene revisar qué contiene el archivo:

```javascript
cat data.txt
```

La salida fue:
```javascript
Gur cnffjbeq vf 7k16JArUVv5LxVuJfsSVdbbtaHGlw9D4
```

A simple vista, esto parece texto alfabético legible, pero no tiene sentido en inglés.
Ese patrón es una pista importante: no estamos ante datos binarios ni Base64, sino ante un texto transformado mediante sustitución de letras.

La estructura:

```javascript
Gur cnffjbeq vf ...
```

se parece mucho a una frase común como:
```javascript
The password is ...
```

Eso sugiere fuertemente que el contenido está en ROT13.

Transformación del contenido con tr

Para revertir ROT13, ejecutamos:
```javascript

tr 'A-Za-z' 'N-ZA-Mn-za-m' < data.txt
```

La salida obtenida fue:

```javascript
The password is 7x16WNeHIi5YkIhWsfFIqoognUTyj9Q4
```

¿Qué es ROT13?

ROT13 significa “rotate by 13”, es decir, rotar 13 posiciones en el alfabeto.

Funciona así:

A se convierte en N
B se convierte en O
C se convierte en P
...
N se convierte en A
O se convierte en B

Lo mismo ocurre con las minúsculas:

a → n
b → o
...
n → a

En otras palabras, cada letra se desplaza 13 posiciones dentro del alfabeto.

Ejemplo rápido

```javascript
hello  -> uryyb
uryyb  -> hello
```

Aquí aparece una propiedad importante:

ROT13 es simétrico.
Aplicarlo una vez transforma el texto; aplicarlo una segunda vez lo devuelve a su forma original.

¿Para qué sirve ROT13?

Aquí hay que ser precisos: ROT13 no es un cifrado seguro.

Sirve principalmente para:

ocultar texto de forma superficial
evitar spoilers casuales
mostrar ejemplos didácticos de sustitución de caracteres

Históricamente se ha usado en foros, listas de correo y comunidades técnicas para ocultar respuestas o contenido que alguien quizá no quiera leer de inmediato.

Lo que no hace

ROT13 no protege realmente la información porque:

no usa clave
siempre aplica la misma transformación
cualquiera que conozca ROT13 puede revertirlo de inmediato

Por eso, técnicamente es más correcto verlo como una ofuscación simple que como cifrado real.

¿Por qué funciona tr 'A-Za-z' 'N-ZA-Mn-za-m'?

El comando tr traduce caracteres de un conjunto a otro.

En este caso:

```jaascript
'A-Za-z'
```

representa todas las letras mayúsculas y minúsculas.

Y:

```javascript
'N-ZA-Mn-za-m'
```

representa esas mismas letras, pero rotadas 13 posiciones.

Desglose
A-Z → todas las mayúsculas
a-z → todas las minúsculas
N-ZA-M → mayúsculas rotadas 13 posiciones
n-za-m → minúsculas rotadas 13 posiciones

La parte:

```jaascript
< data.txt
```


redirige el contenido del archivo como entrada estándar al comando.

Identificación de la contraseña

La salida fue:
```jaascript
The password is 7x16WNeHIi5YkIhWsfFIqoognUTyj9Q4
```

La contraseña del usuario bandit12 es:
```jaascript

7x16WNeHIi5YkIhWsfFIqoognUTyj9Q4
```

Error común en este nivel

Un error frecuente sería aplicar tr sin inspeccionar primero el archivo.
Eso resuelve el reto, sí, pero debilita el razonamiento: ejecutas el comando correcto sin justificar por qué.

Otro error conceptual es asumir que ROT13 ofrece seguridad real.
No la ofrece.

La lección aquí es clara:

inspeccionar primero
inferir el mecanismo
aplicar después la transformación adecuada
Conclusión del nivel

Este nivel introduce una idea importante en Linux y en seguridad:
no toda transformación de texto implica criptografía real.

La solución refuerza varios puntos clave:

inspeccionar el archivo antes de actuar
reconocer patrones compatibles con ROT13
usar tr para revertir la transformación
distinguir entre ofuscación simple y seguridad real

Más allá de obtener la contraseña, este nivel enseña una lección conceptual importante:
si un esquema no usa clave y siempre aplica la misma regla, no ofrece seguridad real.

Con la contraseña obtenida, ya es posible autenticarse como bandit12 y continuar con el Nivel 12 → Nivel 13
