## Bandit – Nivel 5 → Nivel 6
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra en un archivo dentro del directorio `inhere` del usuario **bandit5**.

La particularidad es que el archivo correcto debe cumplir varias condiciones específicas:

- es legible por humanos  
- tiene un tamaño exacto de **1033 bytes**  
- no es ejecutable  

La dificultad principal consiste en no revisar archivo por archivo manualmente, sino en aplicar una búsqueda precisa usando criterios concretos de tamaño, tipo y permisos.

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `ls` – Mostrar el contenido de un directorio  
- `cd` – Desplazarse entre directorios  
- `cat` – Visualizar el contenido de archivos  
- `file` – Obtener información sobre el tipo de archivo  
- `du` – Ver el tamaño de archivos o directorios  
- `find` – Localizar archivos dentro del sistema según distintos criterios  

Aunque hay varios comandos sugeridos, en este nivel el más importante es `find`, ya que permite filtrar archivos según sus características.

---

## Conexión al servidor (Nivel 5)

Para acceder al Nivel 5 de Bandit, nos autenticamos por SSH con el usuario correspondiente a este nivel:

```javascript
sshpass -p '4oQYVPkxZOOEOO5pTW81FB8j8lxXGUQw' ssh bandit5@bandit.labs.overthewire.org -p 2220
```

El uso de sshpass permite pasar la contraseña directamente en el comando, evitando la entrada interactiva y facilitando conexiones automatizadas en un entorno de práctica.

Una vez establecida la conexión, el sistema mostrará el banner de bienvenida del wargame, confirmando que nos encontramos en el entorno correspondiente al Nivel 5.

Ajuste del entorno de terminal

Tras iniciar sesión, se puede definir la variable de entorno TERM para mejorar la compatibilidad de la terminal:

```javascript
export TERM=xterm
```

Este ajuste no es obligatorio para resolver el nivel, pero puede resultar útil en ciertos entornos.

Acceso al directorio objetivo

Desde el directorio personal del usuario bandit5, ingresamos al directorio inhere:

```javascript
cd inhere
```

Aquí se encuentran múltiples subdirectorios y archivos, por lo que revisar todo manualmente sería ineficiente.

Búsqueda del archivo correcto

Para localizar directamente el archivo que cumple con las condiciones indicadas por el nivel, utilizamos:

```javascript
find . -type f -size 1033c ! -executable
```

La salida obtenida es:

```javascript
./maybehere07/.file2
```

¿Por qué funciona este comando?

Este comando utiliza varios filtros a la vez:

find . inicia la búsqueda desde el directorio actual
-type f limita los resultados únicamente a archivos regulares
-size 1033c busca archivos con un tamaño exacto de 1033 bytes
! -executable excluye los archivos que tienen permisos de ejecución
Desglose del criterio

La lógica del nivel consiste en encontrar un archivo que no solo exista dentro de inhere, sino que cumpla varias condiciones simultáneamente.
Aquí, find permite automatizar esa selección sin necesidad de inspeccionar cada archivo a mano.

El sufijo c en 1033c significa bytes.
Esto es importante, porque si se omitiera o se usara otra unidad, la búsqueda devolvería resultados incorrectos.

El operador ! niega la condición que sigue.
En este caso, ! -executable significa: “mostrar solo archivos que no sean ejecutables”.

Obtención de la contraseña del Nivel 6

Una vez identificado el archivo correcto, leemos su contenido con:

```javascript
cat ./maybehere07/.file2
```

La salida revela la contraseña del usuario bandit6:

```javascript
HWasnPhtq9AVKe0dmk45nxy20cvUa6EG
```
