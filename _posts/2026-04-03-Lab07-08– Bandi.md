## Bandit – Nivel 8 → Nivel 9
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío se encuentra en el archivo `data.txt` y corresponde a la **única línea que aparece una sola vez**.

La dificultad principal consiste en distinguir una línea única entre muchas líneas repetidas.  
Por ello, el objetivo es agrupar correctamente las líneas iguales y después filtrar aquella que no tiene duplicados.

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `cat` – Visualizar el contenido de archivos  
- `sort` – Ordenar líneas de texto  
- `uniq` – Filtrar o detectar líneas repetidas  
- `grep` – Buscar texto dentro de archivos  
- `head` – Mostrar las primeras líneas  
- `tail` – Mostrar las últimas líneas  

En este nivel, los comandos clave son `sort` y `uniq`.

Búsqueda de la línea única

Sabemos por el enunciado que la contraseña es la única línea que aparece una sola vez dentro de data.txt.

Para resolverlo, ejecutamos:

```javascript
sort data.txt | uniq -u
```

La salida obtenida es:

```javascript
4CKMh1JI91bUIZZPXDqGanal4xvAg0JM
```

También probamos la forma larga equivalente:

```javascript
sort data.txt | uniq --unique
```

La salida fue exactamente la misma:

```javascript
4CKMh1JI91bUIZZPXDqGanal4xvAg0JM
```

¿Por qué funciona este comando?

La clave está en entender qué hace cada parte del pipeline.

1. sort data.txt

Este comando ordena todas las líneas del archivo.
Eso es importante porque uniq solo compara líneas adyacentes.

Si dos líneas idénticas están separadas en distintas posiciones del archivo, uniq por sí solo no las tratará como duplicadas a menos que antes estén agrupadas una junto a la otra.

2. uniq -u

El comando uniq sirve para trabajar con líneas repetidas consecutivas.

La opción:

```javascript
-u
```

indica que solo deben mostrarse las líneas que aparecen una única vez.

Por tanto, el pipeline completo:

```javascript
sort data.txt | uniq -u
```

La diferencia es solo sintáctica:

-u es la opción corta
--unique es la opción larga y más explícita

En ambos casos, el comportamiento es el mismo: mostrar solo las líneas que aparecen una sola vez.

Conclusión del nivel

Este nivel introduce una habilidad muy importante en Linux:
procesar texto por tuberías y combinar comandos simples para resolver un problema específico.

La solución refuerza varios puntos fundamentales:

entender que uniq trabaja sobre líneas consecutivas
usar sort como paso previo para agrupar duplicados
distinguir entre líneas repetidas y líneas únicas
aprovechar tuberías para encadenar herramientas simples

Más allá de obtener la contraseña, este nivel enseña un patrón clásico en terminal:
preparar los datos primero y filtrarlos después.
