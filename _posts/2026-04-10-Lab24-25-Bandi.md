## Bandit – Nivel 24 → Nivel 25
### Objetivo del nivel

En este nivel, el servicio que escucha en `localhost:30002` pide dos valores en una sola línea:

- la contraseña actual de `bandit24`
- un PIN secreto de 4 dígitos

Como el PIN no lo conocemos, la forma razonable de resolverlo es automatizar todos los intentos posibles desde `0000` hasta `9999` usando un script Bash. En tu sesión, ese enfoque devolvió la contraseña de `bandit25`. :contentReference[oaicite:0]{index=0}

---

## Enfoque de esta guía

Aquí no vamos a resolverlo con un one-liner largo.  
Aquí la idea es hacerlo **solo con un script**, de forma ordenada:

1. crear un directorio temporal
2. entrar en él
3. crear un script
4. darle permisos de ejecución
5. ejecutarlo
6. recuperar la contraseña

---

## ¿Por qué usar un script aquí?

Porque este nivel ya no consiste en un único comando corto, sino en una secuencia lógica:

- leer una contraseña
- generar miles de combinaciones
- enviarlas a un servicio
- filtrar la salida útil

Todo eso se puede hacer en una sola línea, sí, pero en un script queda mejor porque:

- es más legible
- se puede ejecutar de nuevo sin reescribir nada
- deja clara la lógica paso a paso
- es más natural para un proceso repetitivo como una fuerza bruta controlada

---

## Preparación del entorno

Primero creamos un directorio temporal y entramos en él:

```javascript
tmpdir=$(mktemp -d)
cd "$tmpdir"
```

¿Por qué hacer esto?

Porque vamos a crear un archivo auxiliar (pp.sh) y no conviene dejarlo tirado en cualquier sitio.
Un directorio temporal te da:

un espacio limpio
un entorno desechable
orden
facilidad para repetir el procedimiento si algo sale mal

No es estrictamente obligatorio, pero sí es una forma más correcta de trabajar.

Creación del script

Ahora creamos el script completo:

cat > pp.sh <<'EOF'
#!/bin/bash

pass=$(cat /etc/bandit_pass/bandit24)

for pin in $(seq -w 0000 9999); do
    echo "$pass $pin"
done | nc localhost 30002 | grep -v "Wrong"
EOF
Explicación del script
1. Shebang
#!/bin/bash

Le dice al sistema que este archivo debe ejecutarse con Bash.

2. Leer la contraseña actual
pass=$(cat /etc/bandit_pass/bandit24)

Aquí guardamos en la variable pass la contraseña actual del nivel.

¿Por qué hacerlo así?

Porque el servicio pide la contraseña de bandit24 en cada intento.
Si la guardamos en una variable:

evitamos escribirla a mano
reducimos errores
podemos reutilizarla miles de veces dentro del bucle
3. Generar todos los PIN posibles
for pin in $(seq -w 0000 9999); do
    echo "$pass $pin"
done

Aquí ocurre el corazón del nivel.

seq -w 0000 9999 genera todos los números desde 0000 hasta 9999, manteniendo el ancho fijo de 4 dígitos.

¿Por qué importa -w?

Porque el servicio no espera:

1
2
3

sino:

0001
0002
0003

El PIN debe respetar el formato de 4 cifras.

¿Qué genera realmente este bloque?

Líneas como estas:

<password> 0000
<password> 0001
<password> 0002
...
<password> 9999

Es decir, un intento válido por línea.

4. Enviar todos los intentos al servicio
| nc localhost 30002

La tubería manda todas esas líneas al servicio que está escuchando en localhost por el puerto 30002.

Idea importante

Aquí nc no está escuchando; está actuando como cliente y enviando datos.

Y localhost, dentro de la sesión SSH, significa el propio servidor remoto de Bandit.

5. Filtrar la salida útil
| grep -v "Wrong"

La opción -v le dice a grep que muestre las líneas que no contienen la palabra Wrong.

¿Por qué hacerlo?

Porque si no filtras, la salida puede llenarse de respuestas incorrectas y ruido.
Con este filtro te quedas con lo importante:

la línea de bienvenida
Correct!
la contraseña final
Dar permisos de ejecución

Una vez creado el script, hay que volverlo ejecutable:

chmod +x pp.sh
Ejecutar el script

Ahora sí, lo ejecutamos:

./pp.sh

La salida esperada será:

I am the pincode checker for user bandit25. Please enter the password for user bandit24 and the secret pincode on a single line, separated by a space.
Correct!
The password of user bandit25 is iCi86ttT4KSNe1armKiwbQNmB3YJP3q4

Y por tanto, la contraseña del usuario bandit25 es:

iCi86ttT4KSNe1armKiwbQNmB3YJP3q4

Esto coincide con lo que obtuviste en tu sesión.

Script completo listo para copiar y pegar
tmpdir=$(mktemp -d)
cd "$tmpdir"

chmod +x pp.sh
./pp.sh
Qué enseña realmente este nivel

Este nivel no trata de adivinar “el PIN correcto” por intuición.
Trata de reconocer que:

el espacio de búsqueda es pequeño
el formato de entrada es conocido
Bash puede automatizar la generación de intentos
nc puede enviarlos al servicio
grep puede limpiar la salida

En otras palabras:

cuando conoces bien el formato de entrada y el espacio de búsqueda es finito, la automatización es la solución natural.

Errores comunes
1. Probar el PIN manualmente

Es una pérdida de tiempo.
El nivel está hecho para automatizarse.

2. Olvidar -w en seq

Eso rompe el formato de 4 dígitos.

3. Escribir la contraseña a mano

No hace falta; se puede leer directamente del archivo.

4. No filtrar la salida

Sin grep -v "Wrong", la respuesta queda más ruidosa y menos clara.

Conclusión del nivel

Este nivel enseña cómo resolver un problema repetitivo mediante un script Bash simple pero bien planteado.

La solución refuerza varios puntos importantes:

uso de directorios temporales como workspace limpio
creación de scripts reproducibles
uso de variables en Bash
generación masiva de entradas válidas
automatización de fuerza bruta controlada
limpieza de salida con grep
