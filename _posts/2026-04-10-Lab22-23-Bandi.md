## Bandit – Nivel 22 → Nivel 23
### Objetivo del nivel

En este nivel, la contraseña del siguiente desafío no está visible directamente en el directorio personal ni en una ruta fija obvia dentro de `/tmp`.  
La pista central vuelve a estar en un **cron job**, pero esta vez el archivo temporal donde se guarda la contraseña **no tiene un nombre legible o directo**: se calcula dinámicamente a partir del nombre del usuario.

La dificultad principal consiste en leer el script automático, entender cómo construye el nombre del archivo de salida y luego reproducir correctamente ese cálculo para localizar la contraseña de `bandit23`.

La sesión que compartiste muestra exactamente ese flujo: lectura de `/etc/cron.d/cronjob_bandit23`, análisis de `/usr/bin/cronjob_bandit23.sh`, cálculo manual del hash correcto y lectura final del archivo temporal que contiene la contraseña. :contentReference[oaicite:0]{index=0}

---

## Comandos útiles

Para completar este nivel, es posible que necesites apoyarte en los siguientes comandos:

- `cat` – Visualizar el contenido de archivos  
- `echo` – Imprimir texto en la salida estándar  
- `md5sum` – Calcular el hash MD5 de una cadena o archivo  
- `cut` – Extraer columnas o campos de texto  
- variables de shell (`var=valor`) – Guardar resultados intermedios en Bash  

En este nivel, los comandos clave son `cat`, `echo`, `md5sum` y `cut`.

## Conexión al servidor (Nivel 22)

Para acceder al Nivel 22 de Bandit, nos autenticamos por SSH con el usuario correspondiente:

```javascript
sshpass -p 'tRae0UfB9v0UzbCdn9cY0gQnds9GF58Q' ssh bandit22@bandit.labs.overthewire.org -p 2220
```
Una vez dentro, la estrategia correcta vuelve a ser inspeccionar el cron job relevante para el siguiente nivel.
Inspección del cron job

Primero leíste el archivo de cron correspondiente:

cat /etc/cron.d/cronjob_bandit23

La salida fue:

@reboot bandit23 /usr/bin/cronjob_bandit23.sh  &> /dev/null
* * * * * bandit23 /usr/bin/cronjob_bandit23.sh  &> /dev/null

Esto muestra dos cosas importantes:

    el script se ejecuta al reiniciar (@reboot)

    también se ejecuta cada minuto

Además, se ejecuta como el usuario:

bandit23

Todo esto aparece en tu sesión.
¿Qué significa una entrada cron como esta?

La línea:

* * * * * bandit23 /usr/bin/cronjob_bandit23.sh  &> /dev/null

significa:

    * * * * * → se ejecuta cada minuto

    bandit23 → corre con la identidad del usuario bandit23

    /usr/bin/cronjob_bandit23.sh → ese es el script que se lanza

    &> /dev/null → descarta tanto la salida estándar como la de error

Idea importante

Aquí no debes “esperar a que algo mágico pase”.
Debes leer el script y reconstruir su lógica.
Análisis del script

Después inspeccionaste el contenido del script:

cat /usr/bin/cronjob_bandit23.sh

La salida fue:

#!/bin/bash

myname=$(whoami)
mytarget=$(echo I am user $myname | md5sum | cut -d ' ' -f 1)

echo "Copying passwordfile /etc/bandit_pass/$myname to /tmp/$mytarget"

cat /etc/bandit_pass/$myname > /tmp/$mytarget

Este script es el corazón real del nivel. Todo esto aparece explícitamente en tu sesión.
¿Qué hace este script?

Vamos por partes.
1. myname=$(whoami)

myname=$(whoami)

Guarda en la variable myname el nombre del usuario que está ejecutando el script.

Como el cron job corre como bandit23, entonces:

myname=bandit23

Idea importante

Aquí está una trampa conceptual muy frecuente:

Aunque tú estás logueado como bandit22, el script no se ejecuta como tú, sino como bandit23.

Por eso whoami dentro del script devuelve bandit23, no bandit22.
2. Cálculo del nombre del archivo temporal

mytarget=$(echo I am user $myname | md5sum | cut -d ' ' -f 1)

Aquí el script hace varias cosas encadenadas:

    construye una cadena de texto:

    I am user bandit23

    calcula su hash MD5 con md5sum

    usa cut -d ' ' -f 1 para quedarse solo con la parte del hash, sin el resto de la salida

¿Por qué hace esto?

Porque el archivo de salida en /tmp no tiene un nombre legible.
El nombre se deriva matemáticamente a partir de una cadena fija que depende del usuario.

Eso evita que puedas adivinarlo solo mirando el directorio sin entender el script.
3. Mensaje informativo

echo "Copying passwordfile /etc/bandit_pass/$myname to /tmp/$mytarget"

Esto solo imprime un mensaje descriptivo.
No es el paso importante, pero te revela la intención del script:

    origen: /etc/bandit_pass/$myname

    destino: /tmp/$mytarget

Observación importante

Como el job redirige salida a /dev/null, ese mensaje no se ve en ejecución normal del cron.
Solo lo ves al leer el script.
4. Copia final de la contraseña

cat /etc/bandit_pass/$myname > /tmp/$mytarget

Aquí el script hace la acción clave:

    lee la contraseña de /etc/bandit_pass/bandit23

    la escribe en /tmp/<hash_calculado>

Esa es la filtración real que aprovechas para resolver el nivel.
Error útil que apareció en la sesión

Intentaste esto:

mytarget=$(echo I am user $myname | md5sum | cut -d ' ' -f 1)
cat /etc/bandit_pass/$myname > /tmp/$mytarget

y obtuviste:

cat: /etc/bandit_pass/: Is a directory

Esto pasó porque $myname no estaba definido en tu shell actual. Todo esto se ve en tu sesión.
¿Qué significa eso?

Si myname está vacío, entonces:

/etc/bandit_pass/$myname

se convierte en:

/etc/bandit_pass/

y eso no es un archivo, sino un directorio.
Lección importante

No basta con copiar líneas del script.
Debes entender con qué valores se evalúan dentro del contexto real del cron job.

El script no corría con tu sesión actual, sino como bandit23.
Cálculo manual correcto del hash

Luego hiciste lo correcto:

echo "I am user bandit23" | md5sum

La salida fue:

8ca319486bfbbc3663ea0fbe81326349  -

Eso te dio el nombre correcto del archivo temporal. Todo esto aparece en tu sesión.
¿Por qué ese valor?

Porque el script toma exactamente la cadena:

I am user bandit23

y calcula su MD5.
¿Por qué aparece - al final?

md5sum suele devolver:

    el hash

    un espacio

    el nombre del archivo procesado

Cuando recibe entrada estándar, muestra - para indicar justamente eso.

Pero el script usa:

cut -d ' ' -f 1

para quedarse solo con el hash:

8ca319486bfbbc3663ea0fbe81326349

Obtención de la contraseña del Nivel 23

Con el nombre correcto del archivo temporal, lo leíste con:

cat /tmp/8ca319486bfbbc3663ea0fbe81326349

La salida fue:

0Zf11ioIjMVN551jX3CmStKLYqjk54Ga

Por tanto, la contraseña del usuario bandit23 es:

0Zf11ioIjMVN551jX3CmStKLYqjk54Ga

Esto aparece directamente en tu sesión.
¿Qué enseña realmente este nivel?

Este nivel enseña una idea muy importante:

no siempre necesitas adivinar rutas temporales; a veces debes reconstruir cómo fueron generadas.

La enseñanza real no es “usar md5sum” por usarlo.
Es entender esta cadena lógica:

    el cron corre como bandit23

    el script usa whoami

    construye una cadena determinista

    calcula su MD5

    usa ese hash como nombre de archivo

    escribe allí la contraseña

Eso es razonamiento estructural, no prueba y error.
¿Qué es MD5 y por qué aparece aquí?

md5sum calcula un hash MD5, es decir, una función que transforma una entrada en una cadena hexadecimal fija.
Importante

Aquí MD5 no se usa como protección criptográfica seria.
Se usa simplemente para generar un nombre derivado y no obvio de archivo.

Eso significa que el nivel no trata de “romper MD5”, sino de reproducir el mismo cálculo que hace el script.
Error común en este nivel
1. Usar tu propio usuario en lugar del usuario real del script

El script corre como bandit23, no como bandit22.
2. Copiar líneas del script sin entender variables

Eso fue justo lo que provocó que $myname estuviera vacío y que cat intentara leer un directorio.
3. Buscar a ciegas en /tmp

Podrías intentar listar archivos aleatorios, pero el método correcto es derivar la ruta a partir del script.
4. Confundir el hash con la contraseña

El hash MD5 no es la contraseña.
Solo es el nombre del archivo donde termina guardada.
Conclusión del nivel

Este nivel introduce varios conceptos importantes de Linux y seguridad:

    cómo analizar un cron job

    cómo interpretar variables dentro de un script Bash

    por qué el contexto de ejecución (whoami) importa

    cómo se puede derivar una ruta temporal a partir de un hash

    cómo reproducir manualmente la lógica de un script automatizado

La idea central del nivel es esta:

si entiendes cómo un sistema genera una ruta, no necesitas adivinarla.
